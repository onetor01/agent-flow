# Changelog v0.0.78 - v0.0.87

## v0.0.79

- refactor: 精简可选模型列表——`MODELS` 收敛为 `opus[1m]` / `sonnet[1m]` / `qwen3.7-max[1m]` / `glm-5.1[1m]` / `gpt-5.5[1m]`，并同步优化默认 Flow 各节点的模型搭配
- fix: Agent 节点 "Other" 选项无法正确展示——AskUserQuestion 的 `showOther`（默认隐藏）反转为 `hiddenOther`（默认展示，置 true 隐藏），common schema / `buildCodeJSDoc` / CodeExecutor / AskUserQuestionCard 同步
- fix: 画布存在文本选区时 `Ctrl+C` 交还浏览器复制文本，不再劫持为复制节点
- refactor: 优化 silent_task 引导语——自动续轮 / 自动拒绝文案引用 `<completion_contract>` 提示收尾（CompleteTask / TerminateTask）
- refactor: no_input 节点启动按钮初始消息改用 `buildNoInputInitMessage(agent)`，与三端同源引导一致

## v0.0.78

- feat: Code 节点新增第五参数 `askUserQuestion`——异步向用户提问；第六参数 `vscode`——直接调用 VSCode API；函数签名同步为 `run(input, values, runCommand, cwd, askUserQuestion, vscode)`
- feat: 首条用户消息展示当前 Agent 全部可读 shareValues（全量值，不再限制 ≤500 字符过滤），code 节点保持展示完整 shareValues
- feat: AgentOverwrite 机制——code 节点返回 `overwrite` 可临时改写下一个 agent 的 `work_mode` 与 `outputs[].require_confirm`，首条 user 消息上方展示改写提示文本
- fix: 给出理由并拒绝后权限卡片未展示——`answered.message` 补齐到 `ToolPermissionCard` 的 `reason` 字段映射
- fix: 全局 Flow 仅在主动 save 时写磁盘，移除运行时节流回调中的 globalStore.save 自动写入
- refactor: 优化自动拒绝指令——`deny_tools` 拒绝理由补充任务上下文（`依据<task_description>执行任务` 或 `执行用户指定的任务`）
- GetFlowJSONSchema MCP 工具不再对 Agent 暴露 Code 节点 schema
- 优化默认 Flow
